# 📊 Evaluation rubric

Use it for the final project and as a model for evaluating your own students.

| Criterion | Excellent (3) | Satisfactory (2) | Needs improvement (1) |
| -------- | ------------- | ----------------- | ------------------- |
| **Docker** | Builds image, runs container, and explains the layers | Builds and runs with help | Can't run it correctly |
| **Docker Compose** | Brings up app and DB correctly, data persists | Brings it up partially | Doesn't understand the dependencies |
| **CI/CD** | Pipeline runs build/test/image green | Partial pipeline | Doesn't automate |
| **Kubernetes** | Deploys, exposes, scales, and updates | Basic deployment | Can't deploy |
| **DevSecOps** | Runs a scan and interprets the risks | Runs a scan without analysis | Doesn't apply security |
| **Observability** | Inspects logs/metrics and explains their use | Inspects basic logs | Can't diagnose |
| **Final project** | Integrates the full workflow and adapts it to teaching | Integrates a partial workflow | Incomplete submission |
| **Teaching application** | Proposes a clear exercise for their students | Proposes a general idea | Doesn't translate it to class |

## How to score

- 8 criteria × 3 points max = **24 points**.
- **Excellent:** 21-24 · **Satisfactory:** 14-20 · **Needs improvement:** < 14.

## Relationship to gamification

| Result | Equivalent to |
| --------- | ---------- |
| 21-24 pts | 🏛️ DevOps Architect |
| 14-20 pts | ☸️ Kubernetes Pilot |
| < 14 pts | keep stacking up missions 💪 |

> The rubric measures **competence**; gamification measures **progress**. Together they keep the motivation going.
