# 📊 Rúbrica de evaluación

Úsala para el proyecto final y como modelo para evaluar a tus propios alumnos.

| Criterio | Excelente (3) | Satisfactorio (2) | Requiere mejora (1) |
| -------- | ------------- | ----------------- | ------------------- |
| **Docker** | Construye imagen, ejecuta contenedor y explica las capas | Construye y ejecuta con apoyo | No logra ejecutar correctamente |
| **Docker Compose** | Levanta app y DB correctamente, datos persisten | Levanta parcialmente | No comprende las dependencias |
| **CI/CD** | Pipeline ejecuta build/test/imagen en verde | Pipeline parcial | No automatiza |
| **Kubernetes** | Despliega, expone, escala y actualiza | Despliegue básico | No logra desplegar |
| **DevSecOps** | Ejecuta escaneo e interpreta riesgos | Ejecuta escaneo sin análisis | No aplica seguridad |
| **Observabilidad** | Consulta logs/métricas y explica su uso | Consulta logs básicos | No logra diagnosticar |
| **Proyecto final** | Integra el flujo completo y lo adapta a docencia | Integra flujo parcial | Entrega incompleta |
| **Aplicación docente** | Propone práctica clara para sus alumnos | Propone idea general | No aterriza a clase |

## Cómo calcular

- 8 criterios × 3 puntos máx = **24 puntos**.
- **Excelente:** 21-24 · **Satisfactorio:** 14-20 · **Requiere mejora:** < 14.

## Relación con la gamificación

| Resultado | Equivale a |
| --------- | ---------- |
| 21-24 pts | 🏛️ Arquitect@ DevOps |
| 14-20 pts | ☸️ Pilot@ de Kubernetes |
| < 14 pts | sigue sumando misiones 💪 |

> La rúbrica mide **competencia**; la gamificación mide **progreso**. Juntas mantienen la motivación.
